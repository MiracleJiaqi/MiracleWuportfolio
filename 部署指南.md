## 一、服务器架构说明

### 当前架构
```
宿主机（36.151.148.84）
├── /PersonalWeb/dist/          ← GitHub Actions部署到这里
│   ├── public/                 ← Nginx根目录
│   │   ├── index.html
│   │   └── assets/*.js, *.css
│   └── index.js
│
└── Docker容器（GSManager3）
    ├── 挂载：-v /PersonalWeb:/PersonalWeb
    ├── 端口：-p 80:80
    └── Nginx监听80端口 → /PersonalWeb/dist/public
```

### 关键点
- **文件传输**：GitHub Actions → 宿主机 `/PersonalWeb/dist/`
- **Nginx位置**：运行在Docker容器内
- **文件访问**：容器通过挂载访问宿主机文件
- **SSH登录**：默认进入容器，需要 `exit` 才能到宿主机

## 三、问题排查指南

### 问题1：GitHub Actions成功但网站没更新

**可能原因**：
- 浏览器缓存

**解决方法**：
```bash
# 强制刷新浏览器
Ctrl + Shift + R（Windows）
Cmd + Shift + R（Mac）
```

### 问题2：部署失败（SSH错误）

**检查步骤**：
1. 确认GitHub Secrets中 `SSH_PRIVATE_KEY` 正确
2. 在服务器上测试SSH连接：
   ```bash
   ssh root@36.151.148.84
   ```

### 问题3：文件上传了但看不到

**原因**：你在容器内，文件在宿主机

**解决**：
```bash
# 如果在容器内（提示符：root@815d2e55c0b7）
exit

# 退出到宿主机（提示符：root@lavm-jm82xzr3zj）
ls -la /PersonalWeb/dist/public/
```

### 问题4：Nginx没有重载

**手动重载**：
```bash
# 在宿主机上执行
docker exec GSManager3 nginx -s reload
```

## 五、关键注意事项

## 六、紧急恢复

### 如果容器意外停止
```bash
# 启动容器
docker start GSManager3

# 进入容器启动Nginx
docker exec -it GSManager3 bash
nginx
exit
```

### 如果容器被删除
```bash
# 重新创建容器（使用原镜像）
docker run -d --name GSManager3 \
  -p 80:80 \
  -v /PersonalWeb:/PersonalWeb \
  -v gsmanager_gsm3_data:/root/server/data \
  -v /opt/GSManager/game_file:/root/steam/games \
  --restart unless-stopped \
  172.16.0.9/xiaozhu674/gameservermanager:latest

# 进入容器安装Nginx
docker exec -it GSManager3 bash
apt update
apt install nginx -y

# 配置Nginx
cat > /etc/nginx/sites-available/portfolio << 'EOF'
server {
    listen 80;
    server_name _;
    root /PersonalWeb/dist/public;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
EOF

rm -f /etc/nginx/sites-enabled/default
ln -s /etc/nginx/sites-available/portfolio /etc/nginx/sites-enabled/
nginx
exit
```

## 八、常见问题FAQ

**Q: 为什么我SSH登录后看不到 `/PersonalWeb` 文件？**
A: 你在容器内，输入 `exit` 退出到宿主机。

**Q: 如何确认部署是否成功？**
A:
1. GitHub Actions显示绿色✓
2. 在服务器上：`ls -lt /PersonalWeb/dist/public/assets/ | head` 查看文件时间
3. 浏览器强制刷新后能看到新内容

**Q: 部署后网站显示404？**
A:
1. 检查Nginx是否运行：`docker exec GSManager3 ps aux | grep nginx`
2. 检查文件是否存在：`ls /PersonalWeb/dist/public/index.html`
3. 重载Nginx：`docker exec GSManager3 nginx -s reload`

**Q: 可以直接在服务器上修改文件吗？**
A: 不推荐！下次GitHub部署会覆盖你的修改。应该在本地修改后push。

**Q: 如何回滚到上一个版本？**
A:
```bash
git log --oneline          # 查看提交历史
git reset --hard <commit>  # 回滚到指定提交
git push -f origin master  # 强制推送（触发部署）
```
